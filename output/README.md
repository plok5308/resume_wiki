# output

생성된 자기소개서 결과물 보관소.

## 파일 세트
한 번 생성할 때 같은 이름으로 **두 개** 파일을 만든다:
- `{stem}.md` — 정본 (메타 주석 포함, 추후 wiki 역반영의 근거)
- `{stem}.txt` — 평문 (지원 시스템 붙여넣기용, 메타·마크다운 제거)

## 파일명 규칙
`{YYYY-MM-DD}-{회사}-{직무}` 를 stem으로 사용
예: `2026-04-27-kakao-ml-engineer.md` / `.txt`

## .md 구조
```
<!--
generated_at: 2026-04-27
job_source: sources/jobs/2026-04-kakao-ml-engineer.md
wiki_used:
  - wiki/strengths.md#문제정의력
  - wiki/experiences.md#카카오톡-알림-개선-프로젝트
  - wiki/writing_style.md (샘플 1, 2)
-->

# {회사} {직무} 자기소개서

(소제목 없이 본문만, 지원동기 → 경력 → 포부 순서로 단락만 분리)
```

## 기본 흐름 (소제목 없는 메일형이 기본)
공고에 별도의 문항 명세가 없으면 **소제목 없이** 다음 3단 흐름을 단락 분리만으로 표현한다.
1. **지원동기** (1~2 단락) — 산업·도메인 관점의 한 줄 명제 + 본인 정체성 한 줄 + 회사 특성과 본인 경험을 직접 연결한 단락
2. **경력** (2~5 단락) — 시간순(박사 → 삼성 → 현대차) 또는 도메인순으로 경험 카드 나열, 결과 단락에 수치·수상·학회 집중
3. **포부** (1 단락, = 마무리) — 회사·직무에 대한 합류 의지, 장기 성장 톤. "감사합니다. / 박진석 드림"으로 종결

`# 제목` 한 줄(자기소개서 헤더)만 두고 본문 내부에는 `## 지원동기` 같은 소제목을 넣지 않는다.
공고에 입력 문항이 분리돼 있으면 문항을 그대로 헤더로 쓰되, 같은 3단 흐름을 문항 안에서 재현한다.

## .txt 변환 규칙
- 메타 주석(`<!-- -->`) 제거
- 마크다운 헤더(`#`)는 제거하고 줄바꿈 두 번으로 분리
- 굵게/이탤릭(`**`, `*`) 마크업 제거, 텍스트만 남김
- 인코딩: UTF-8

## 역반영
사람이 다듬은 결과 중 재사용 가치 있는 표현·일화는 wiki에 반영.
특히 새로 등장하거나 수정된 문구는 `wiki/writing_style.md`의 원문 샘플에 추가.
